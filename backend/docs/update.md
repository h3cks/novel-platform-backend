# Інструкція з оновлення системи

## 1. Підготовка до оновлення
* **Створення резервних копій:** Завжди створюйте бекап бази даних перед оновленням (див. `backup.md`).
* **Планування простою:** Оновлення зазвичай займає до 2-х хвилин. Сповістіть користувачів у разі масштабних міграцій БД.

## 2. Процес оновлення
Виконайте наступні команди у директорії проєкту:
\`\`\`bash
# 1. Завантаження нового коду
git pull origin main

# 2. Оновлення залежностей
npm ci

# 3. Міграція бази даних (якщо є зміни у схемі)
npx prisma migrate deploy
npx prisma generate

# 4. Збірка нового коду
npm run build

# 5. Перезапуск процесу без простою
pm2 reload novel-api
\`\`\`

## 3. Процедура відкату (Rollback)
У разі виявлення критичних помилок після оновлення:
1. Визначте хеш попереднього стабільного коміту.
2. Виконайте відкат:
   \`\`\`bash
   git checkout <commit_hash>
   npm ci
   npx prisma generate
   npm run build
   pm2 reload novel-api
   \`\`\`
3. Відновіть БД з бекапу, якщо міграції пошкодили дані.
